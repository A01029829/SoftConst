Flujo de trabajo de Git y GitHub

Página de inicio del blog

Descripción del flujo:

El flujo de trabajo en Git comienza clonando el repositorio con git clone. Luego, se realizan cambios en los archivos y se revisan con git status. Los cambios se agregan al área de staging con git add y se confirman con git commit -m "mensaje", guardando así el progreso en el repositorio local.

Pasos del flujo de trabajo:

  1. Clonar el repositorio
  2. Crear o modificar archivos
  3. Verificar el estado del repositorio
  4. Agregar cambios al área de staging
  5. Confirmar los cambios con un commit
  6. Subir los cambios al repositorio remoto
